微信微博整合平台数据库设计草稿

By 张伦

Published 2017-04-23

  2. 2. 子表:任务爬取(Request_info),爬取数据(Crawl_db)

主体分三个部分，最重要部分关键字，存贮爬取关键字，剩下两个部分直接与关键字相连（目前采用外键强连接），分别是关键字请求记录（记录关键字请求数据），关键字爬取数据（记录由关键字爬取过来的数据）

主表：关键词（tag）

主要字段

  • name 关键字名字,便于人工管理
  • key 关键字
  • wid 关键字限制条件,比如说限定某个公众号、特定时间等
  • mode 关键字模式,区分爬取类型,比如说爬取微博博主、公众号、公众号历史文章等
  • level 关键字等级,区分爬取频率,也用来人工标注供机器模型构造

子表:任务爬取(Request_info),爬取数据(Crawl_db)

任务爬取表(Request_info)

主要字段

  • tag_id 关联关键字id,用来外键绑定
  • start_time 任务开始时间
  • end_time 任务结束时间
  • create_sum 任务新建总数,如新插入多少微博博主,多少公众号,多少文章等
  • update_sum 任务更新总数,比如更新多少已收录微博博主微博等

爬取数据表(Crawl_db)

主要分四个表,还有一些子表,比如微博和公众号文章,标题、正文、发表时间不会改变，但是转发量、点赞量会随时间改变二改变，所以用子表把会随时间改变的分离出去。

  • 微博账号表
  • 微博数据表
    • 微博转发评论点赞表
  • 公众号账号表
  • 公众号文章数据表
    • 公众号文章评论、正文表

四个表都有一个tag_id字段与主表关键字相连,具体字段由爬取数据决定,基本上能爬取到的数据都会由相应字段.
